# arXiv Keyword Pin - 关键词置顶高亮

## 简介
`arXiv Keyword Pin` 是一个用于 arXiv 论文列表页面的浏览器用户脚本（UserScript）。该脚本的核心机制是根据用户自定义的关键词，自动检索当前页面的文章（匹配范围包括标题、作者与摘要），将命中的文章置顶显示，并对命中的关键词进行高亮标注。此外，脚本重构了原生页面的 DOM 结构，引入了卡片式布局、暗色模式以及与 INSPIRE-HEP 数据库的快捷链接。

## 核心功能说明

* **关键词匹配与置顶重排**
    * 支持多行关键词输入，对文章的 Title（标题）、Authors（作者）和 Abstract（摘要）进行匹配。
    * 将所有匹配的文章统一提取并置顶排序在页面最前端，未匹配的文章排列其后。
    * 支持普通字符串匹配以及**正则表达式 (Regex)** 匹配。
    * 可自由切换**区分大小写**与**不区分大小写**模式。
* **摘要自动展开与异步加载**
    * 自动触发页面上被折叠的摘要节点。
    * 针对部分未完整加载摘要的文章（新版或特定列表页），脚本会通过后台并发发起 AJAX 请求，提取详情页的完整摘要并注入回列表页，确保关键词匹配的完整性。
* **卡片化网格布局**
    * 提供原生列表模式与卡片模式的切换。
    * 在卡片模式下，文章以外部独立卡片形式呈现，支持配置 1 至 5 列的响应式网格布局。
* **视觉与交互增强**
    * **关键词高亮**：对匹配成功的文本节点进行背景高亮。为避免破坏 LaTeX 公式渲染，高亮逻辑会自动避开包含 MathJax 的节点。
    * **暗色模式 (Dark Mode)**：内置独立的深色主题，覆盖原生背景、卡片样式及字体颜色。
    * **悬浮控制面板**：页面右侧生成一个可任意拖拽的悬浮按钮（⚙），点击可呼出可视化配置面板，无需修改代码即可实时调整所有参数。
    * **匹配统计计数器**：在悬浮按钮旁实时显示当前页面 `匹配文章数 / 总文章数` 的统计标签。
* **第三方数据库联动**
    * 自动解析文章作者列表，在每位作者姓名后追加一个快捷按钮 (`I`)。点击后将跳转至 `INSPIRE-HEP` 数据库，按作者姓名进行精准检索。
* **MathJax 公式重排兼容**
    * 针对由于 DOM 节点移动导致的 LaTeX 公式丢失或乱码问题，脚本内置了对 MathJax v2 与 v3 的兼容逻辑。在元素重排后，自动调用 MathJax 引擎对未渲染的 TeX 文本块进行重新排版。

## 安装指南

1.  **安装脚本管理器**：
 [Tampermonkey](https://www.tampermonkey.net/) 浏览器扩展（[Chrome](https://chrome.google.com/webstore/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo) / [Firefox](https://addons.mozilla.org/en-US/firefox/addon/tampermonkey/) / [Edge](https://microsoftedge.microsoft.com/addons/detail/tampermonkey/iikmkjmpaadaobahmlepeloendndfphd)）
2.  **安装本脚本**：
    新建一个脚本，将本项目的源代码完整复制并粘贴到编辑器中保存。或者通过脚本管理器的 URL 导入功能进行安装。
3.  **生效范围**：
    脚本默认匹配以下 URL 规则：
    * `https://arxiv.org/list/*`
    * `https://arxiv.org/list?*`

## 使用与配置方法

脚本的所有功能均可通过页面上的**悬浮控制面板**进行控制。

### 1. 悬浮面板操作
* **拖拽**：鼠标左键长按悬浮按钮（⚙）即可在页面可视区域内自由拖放，脚本会自动记忆最后一次的坐标位置。
* **打开/关闭**：单击悬浮按钮，或在脚本管理器的菜单中点击 `🏷️ 设置关键词`。

### 2. 参数设置选项
在悬浮面板中，可配置以下参数：

* **关键词输入框**：
    * 按行输入关键词，每行代表一个独立的匹配项。
    * 例如：输入 `dark matter` 和 `gravitational wave`（分两行），脚本将匹配包含任意一个词汇的文章。
* **卡片模式 (Checkbox)**：开启后使用现代化卡片包裹每篇文章；关闭则保留 arXiv 原生 `<dl>`/`<dt>`/`<dd>` 列表样式。
* **区分大小写 (Checkbox)**：开启后严格按照输入的大小写进行匹配（例如 `Axion` 不会匹配 `axion`）。
* **正则模式 (Checkbox)**：开启后，输入框内的文本将被解析为 JavaScript 正则表达式。此时可以使用 `\b`（单词边界）、`|`（或）等高级语法。注意：开启此模式需确保输入的正则语法正确，否则该行将被忽略。
* **深色模式 (Checkbox)**：全局切换为暗色背景（依赖卡片模式）。
* **排版列数 (Dropdown)**：支持 1 列、2 列、3 列、4 列、5 列切换，建议在大屏显示器下使用多列模式以提高阅读效率。

### 3. 应用与生效
在面板中调整完参数后，点击底部的 **应用** 按钮。脚本会保存当前配置至浏览器本地存储（Local Storage），并自动刷新页面以应用最新规则。

## 技术细节与数据流向
* **数据存储**：所有配置项（包括悬浮按钮的坐标）均通过 `GM_getValue` 和 `GM_setValue` 储存在本地，不会跨设备同步。
* **异步请求限制**：由于自动抓取摘要涉及网络请求，脚本内置了 `MAX_CONCURRENT = 5` 的并发数控制，以避免对 arXiv 服务器造成瞬时过大压力并防止 IP 被临时封禁。
